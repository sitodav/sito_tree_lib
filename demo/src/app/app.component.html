Tree example: click on a white area to create a node. Drag a node onto another to merge them. Double click to explode and see the childs.


Example 1: Interactive, no autosize, color by cluster (nodes get the same color of the father/cluster).
Click on the canvas to create a new node
Drag a node on top of another (on a different cluster) to append to, double click a node to expand. Click on an edge to select it.

Example 2: Similar as before (but non-clickable edges), but node get bigger the more children they have

Example 3: Custom graphical props. Readonly, autosize, color by state (the color of a node depends on its internal status).
The tree is non modifiable, and it's nodes are loaded asynchronously (after some second) from read (mocked) data

Double click a node to expand. Click update button to change node state/color   , or expand  

Example 4: Readonly, color by state, but with multiple fathers for a node allowed. Clickable edges, but with no callback on click

Example 5: Same as before (but clickable edges with callback for setting highlight and click handler on nodes for highlight on those too), but trees are disposed along row-wise using a fixed layout.

Example 6: Same as before, but node are disposed using column-wise layout

Example 7: Interactive tree with registered callback.
Open the console to see the logs on double click and node creation (there are lots of events you can catch)
Example 8: Tree loaded via json and interactive, with data exporting in json. You can configure what json schema to use and what data use as node status.
Interact to create new nodes on the loaded (via json data) tree. On every interaction with the tree (create/append etc) the data is exported (on the right)
